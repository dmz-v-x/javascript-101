## Immediately Invoked Function Expression (IIFE)

## 1. What is an IIFE?

IIFE stands for **Immediately Invoked Function Expression**.

Simple definition:

> An IIFE is a function that runs immediately after it is created.

So instead of:
- defining a function
- then calling it later

We:
- define the function
- call it instantly

---

## 2. Why Does IIFE Exist?

IIFE exists to solve two main problems:

1. **Avoid polluting the global scope**
2. **Create private variables**

Before modern JavaScript features existed, IIFE was the main way to:
- isolate code
- protect variables
- avoid name conflicts

---

## 3. Basic Syntax of IIFE

The most common syntax:

    (function () {
      console.log("IIFE executed");
    })();

Break it down:
- `function () {}` → function expression
- `( )` around function → tells JS it’s an expression
- final `()` → immediately calls the function

---

## 4. How IIFE Works (Step by Step)

Example:

    (function () {
      console.log("Hello");
    })();

What happens:
1. JavaScript sees a function expression
2. Function is created
3. Function is immediately executed
4. Code inside runs once
5. Function is discarded

Important:
- You cannot call it again
- It runs only once

---

## 5. IIFE with Parameters

IIFE can accept arguments.

Example:

    (function (name) {
      console.log("Hello", name);
    })("Asha");

Explanation:
- `name` is parameter
- `"Asha"` is argument
- value flows just like a normal function

---

## 6. IIFE with Return Values

IIFE can also return values.

Example:

    const result = (function () {
      return 10 + 20;
    })();

Result:
- `result` → 30

Important:
- returned value can be stored
- function itself is gone after execution

---

## 7. Scope and Data Privacy (VERY IMPORTANT)

Variables inside an IIFE are **private**.

Example:

    (function () {
      let secret = "hidden";
      console.log(secret);
    })();

    console.log(secret); // ❌ Error

Why?
- `secret` exists only inside IIFE
- global scope is protected

This was extremely important before:
- `let`
- `const`
- modules

---

## 8. Different IIFE Syntax Variations

All of these are valid IIFEs:

    (function () { })();

    (function () { }());

    (() => {
      console.log("Arrow IIFE");
    })();

They all:
- create a function expression
- invoke it immediately

---

## 9. Common Beginner Mistakes

- Forgetting parentheses
- Trying to call IIFE again
- Expecting IIFE to be reusable
- Writing large logic inside IIFE
- Confusing IIFE with normal functions
- Forgetting semicolon before IIFE (important!)

---

### ⚠️ Semicolon Gotcha

If previous code does not end with `;`:

    const x = 10
    (function () { })();

This can break code.

Safe practice:

    ;(function () { })();

---

## 10. Points to Remember

- IIFE runs immediately
- It is a function expression
- Used for one-time execution
- Creates private scope
- Helps avoid global pollution
- Cannot be reused
- Mostly historical but still useful
